1. Executive Summary
1.1 Product Vision
OFAS (Orchestrated Financial Analysis System) is an autonomous multi-agent AI system that replicates the complete workflow of a professional M&A/PE deal team. The system ingests financial data, builds institutional-grade Excel models with formulas intact, conducts valuation analysis, performs due diligence, and generates client-ready deliverables (IC memos, pitch decks, research notes) with full audit trails and citations[1][2][3].
1.2 Business Objectives
Replace 80+ hours of manual analyst work per deal with 4–6 hours of AI-orchestrated execution
Maintain institutional quality standards: Excel models with formulas, not hard-coded values; auditable assumptions with RAG citations
Enable scalability: Process 10–20 deals simultaneously vs. 2–3 with human teams
Reduce errors: Automated compliance checks, balance sheet reconciliation, citation validation

2.3 Workflow Loop (Mission Control Pattern)
Execution Flow:
Supervisor receives objective (e.g., "Evaluate acquisition of TargetCo for $500M")
Creates tasks with RACI matrix:
Task 1: Data ingestion → Librarian (R), Supervisor (A)
Task 2: Build 3-statement + DCF → Architect (R), Supervisor (A), Valuation (C)
Task 3: Comps analysis → Valuation (R), Supervisor (A), Architect (C)
Task 4: Risk DD → M&A Agent (R), Supervisor (A)
Task 5: IC memo → Reporting (R), Supervisor (A), all others (C)
Task 6: Compliance review → Compliance (R), Supervisor (A)
Sequential execution with feedback:
Each agent updates task status: pending → running → done or blocked
Issues logged with severity (info, warn, error) and ownership
If blocked or error: Supervisor routes to responsible agent for fix
Quality gates:
After Architect: Compliance validates balance sheet, cash ties
After Reporting: Compliance validates citation coverage, number consistency
Only when all tasks done and no error issues → mission_status: ready_for_client

3.2.2 Model Population Process
Step 1: Template Selection
Based on deal type:
Standard corporate → 3-statement + DCF
M&A → Accretion/Dilution + 3-statement
SaaS → SaaS forecasting + DCF
PE/VC → Cap table + waterfall
Step 2: Input Mapping
Load template using openpyxl[13]
Identify input cells (typically color-coded blue or named ranges)
Map API data to input cells:
Example: Populate revenue inputs
ws_inputs = wb['Assumptions']ws_inputs['B5'] = financial_data['revenue'][2021] # Historicalws_inputs['B6'] = financial_data['revenue'][2022]ws_inputs['B7'] = financial_data['revenue'][2023]ws_inputs['B8'] = 0.12 # Growth assumption
Step 3: Formula Preservation
CRITICAL: Never overwrite formula cells[1][2]
Only write to designated input/assumption cells
All calculation cells retain original formulas:
Revenue forecasts: =B7 * (1 + B8)
EBITDA: =Revenue - COGS - OpEx + D&A
Free Cash Flow: =EBITDA * (1 - TaxRate) - CapEx - ΔWC
Enterprise Value: =NPV(WACC, FCF_range) + TerminalValue
Step 4: Assumption Documentation
Add cell comments with RAG citations:
from openpyxl.comments import Comment
comment = Comment("Source: 10-K 2023, MD&A, page 35\n""Historical CAGR: 11.2%\n""Management guidance: 10-15% through 2026\n""RAG chunk_id: msft_2024_chunk_847","OFAS Architect Agent")ws_inputs['B8'].comment = comment
Step 5: Quality Checks
Balance sheet balances: Assets = Liabilities + Equity
Cash flow ties: Beginning Cash + CFO + CFI + CFF = Ending Cash
P&L to cash flow reconciliation
All checks logged as issues if fail

Data Collection
Identify peers based on:
Sector classification (GICS, SIC)
Geography and market cap
Business model similarity
Fetch metrics via API:
Market cap, enterprise value
Revenue, EBITDA, EBIT, Net Income
Multiples: EV/Revenue, EV/EBITDA, P/E, P/B

Compute quartiles (25th, median, 75th percentiles)

Calculate implied valuation range

3.7.2 Issue Logging
Every check failure creates an Issue object:
{"msg": "Balance sheet does not balance: Assets 1,250 vs. L+E 1,248","owner": "architect","severity": "error","task_id": "model","timestamp": "2026-03-04T22:15:33Z"}

warn severity allows completion but flags for review
info severity is informational only
3.7.3 Feedback Loop
If issues exist after agent completes task:
Supervisor routes back to responsible agent
Agent re-runs with issue context
Maximum 3 retry cycles before escalating to human
